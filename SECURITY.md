# Security Policy: Project AEGIS

## 🛡️ Responsible Disclosure
As a research prototype dedicated to integrity and asset protection, Project AEGIS takes security seriously. If you discover a vulnerability within this framework or a method to bypass the proposed Tier 2/3 mitigations, please report it responsibly.

### Reporting a Vulnerability
* **Preferred Method:** Open a "Security Inquiry" via GitHub Issues or contact the maintainer directly.
* **Response Time:** Every effort will be made to acknowledge reports within 48–72 hours.

## ⚖️ Scope & Liability
* **Research Status:** Project AEGIS is a **Conceptual Framework** and **Research Prototype**. 
* **No Warranty:** The code and methodologies provided are for educational and strategic simulation purposes. The author is not liable for any system instability or data loss resulting from the implementation of these experimental architectures.
* **Non-Malicious Use:** This project is designed to *defend* digital ecosystems. Any use of these methodologies to develop "anti-anti-cheat" or offensive exploits is strictly prohibited and contrary to the author's intent.

## 🔐 Data Privacy
This project adheres to **Privacy-by-Design** principles. All provided utilities (e.g., `scrub_pii.py`) are intended to ensure compliance with the **EU AI Act (2026)** and **GDPR**.
